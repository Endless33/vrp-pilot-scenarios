# Pilot Non-Disclosure Expectations

## Purpose

VRP pilot evaluations are intended to validate observable behavior.

The objective is evaluating runtime outcomes.

The objective is not transferring protected implementation knowledge.

---

## Scope

Pilot participants may receive:

- runtime artifacts
- evaluation binaries
- validation outputs
- pilot reports
- evidence artifacts

Pilot participants should not expect access to:

- private VRP core source code
- protected recovery mechanisms
- internal authority algorithms
- implementation-specific runtime logic

---

## Evaluation Boundary

Pilot evaluation focuses on:

```text
observable behavior

runtime evidence

validation outcomes

correctness properties
```

The focus is:

```text
what the system does
```

not:

```text
how the private core does it
```

---

## Intellectual Property

VRP contains proprietary research and development.

Pilot participation does not imply:

- source code transfer
- ownership transfer
- implementation rights
- derivative work rights

Unless explicitly agreed otherwise.

---

## Evidence Sharing

Pilot participants may discuss:

- evaluation results
- observed behavior
- operational findings
- pilot conclusions

Pilot participants should avoid disclosure of:

- confidential pilot materials
- protected implementation details
- non-public evaluation artifacts

---

## Responsible Evaluation

The objective is technical validation.

Not reverse engineering.

Not implementation extraction.

Not competitive replication.

---

## Core Observation

A pilot should provide enough visibility to evaluate behavior.

A pilot does not require disclosure of protected implementation mechanisms.

Observable correctness and protected implementation can coexist.
